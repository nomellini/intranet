CheckList para Revisão de Forms

 Nome:  Responsável:
 Data de Início:  Hora de Início:
 Data de término:  Hora de Término:

 

   1. Verifique se todos os componentes estão com hints associados corretamente
   2. Verifique se a frase do hint está no padrão Xxxx xxx, ou seja, a primeira letra da primeira palavra com letra maiúscula e o resto em minúsculo. Exemplo : Visualizar impressão
   3. Quando houver Shortcut no componente, verifique se a frase do hint mostra este shortCut. No uso de ActionList, isto é feito automaticamente.
   4. Verifique se é possível criar ActionList ao invés de trabalhar com eventos de execução OnClick diretamente no componente 
   5. Verifique se os botões do DBNavigator operam Enable/Disable de forma correta.
   6. Verifique se os campos chave em um form de cadastro estão em negrito e na cor clSilver.
   7. Verifique se ao colocar em modo de inserção, estes campos chaves ficam na cor normal, ou seja branco.
   8. Verifique se na inclusão, não é possível incluir uma chave que já exista, garantindo a consistência da base.
   9. Verifique se não há campo checkbox ao invés de radiobutton.
   10. Verifique se não há componentes que podem ser substituídos por controles mais fáceis como checkbox, radiobutton, combobox,etc. Por exemplo, você tem um radiogroup com opções: Mensalista, horista, todos. Você pode trocar isto por dois checkbox Mensalista, horista.
   11. Verifique se a pesquisa nos forms de cadastro funcionam corretamente.
   12. Verifique se o título nos grids estão com o nome correto
   13. Verifique se ao abandonar um form e o usuário estiver editando um registro, há uma mensagem esclarecendo que ele irá perder os dados se fizer isto.
   14. Verifique se o botão de pesquisa dos forms estão no padrão. glyph correto - binóculo, posição relativa no form (em cima, a direita).
   15. Nos forms onde tenha statusbar, verifique se as mensagens estão corretas em diferentes situações
   16. Verifique o português de forma rígida. Sistema com palavra errada é coisa de amador.
   17. Verifique se as teclas de atalho funcionam corretamente. Elas estão definidas mais abaixo. Teclas de atalho para forms de cadastro Salvar - Ctrl B Novo - Ctrl A Refresh - F5 Teclas de atalho para forms de configuração de relatórios Refresh - F5 Imprimir - Ctrl P
   18. Sempre minimize a disposição dos componentes para deixar o form o menor possível. Verifique os panels e ajuste os componentes de forma a tornar o panel o menor possível, levando em conta sempre a legibilidade.
   19. Verifique se os lookups estão trazendo a informação de forma correta.

-

   20. Verifique se os labels estão corretamente dispostos para cada edit, seguindo o padrão de maiúsuculas e minúsculas.
   21. Se for um relatório, verifique se a padronização dos relatórios esta sendo respeitada: -Alinhamento dos campos -(Mask valor) Alinhamento a direita.
   22. Verifique se na Query quando for campo currency foi adotar no display format a mascara (###,###,##0.00).
   23. Verifique se a DbGrid esta em read-only, DbGrid não pode por a tabela em modo de inserção.
   24. Verifique se o OnClick do Navigator puxa o foco para o primeiro Edit.
   25. Verifique se o After Post da Table (fecha e abre) atualiza a Table.
   26. Verifique se os componentes de data(dd/mm/aaaa, mm/aaaa ou de data até data) podem ser substituidos pelos novos componentes BmDateEdit, BmDbDateEdit e BmDatefromto.
   27. Testar se a tabela esta indexada(F5 - Refresh).
   28. Verifique se o TabOrder esta correto.
   29. Verifique se nos componente BmDateEdit, BmDbDateEdit e BmDatefromto os nomes dos dias da semana (DayNames) estão no padrão 2a, 3a, 4a, 5a, 6a, Sa e Do.
   30. Verifique se nos forms de configuração de relatório o parametro WindowState esta como wsMaximized.
   31. Verifique se nos forms de Cadastro o item BorderStyle esta como BsSingle.
   32.Testar os botões de Atualizar (que estão chamando a mesma rotina no ActionList) e ver se todos os campos do relatório aparecem em tela.
   33.A propriedade GRID OPTIONS de qualquer tabela deve estar nesta sequência
False, False, True, True, False, True, True, False, True, False, True, True, False
