Projetos : Cobol Web

Sistemas Datamace via Web

 

Introdução

Torna-se pouco necessária a argumentação a respeito da necessidade de prepararmos nossos sistemas para o acesso via Intranet/Internet. A revolução causada por essas novas formas de acesso não deixam dúvidas quanto à sua importância. Tornar um sistema disponível para Intranet/Internet significa que o mesmo possa ser acessado via browser (navegador), normalmente o Internet Explorer ou o Netscape. Da mesma forma como acessamos o Internet Banking e fazemos as nossas movimentações, por exemplo. Até algum tempo atrás, somente outras linguagens de programação e outros formatos de arquivos (não Cobol) podiam ser acessados via Intranet/Internet. A partir da versão 2.0, o NetExpress (que é uma ferramenta para desenvolvimento em Cobol), incorporou recursos que possibilitam a execução de programas feitos em Cobol em ambiente Web, e assim, todos formatos de arquivos nativos do Cobol (como os que nós utilizamos) podem ser acessados.

Conceitos

Quando acessamos uma página na Web, significa que "do outro lado" da linha existe um servidor. Mais precisamente um servidor Web, o qual envia a página solicitada por nós para o nosso navegador, que por sua vez interpreta os comandos escritos na linguaguem HTML (Hyper Text Markup Language) e resulta no visual o qual já estamos acostumados a ver na Internet. Os comandos em HTML que geram a página podem ser fixos, ou seja, estarem gravados num arquivo no servidor Web e, o qual quando solicitado, envia um conteúdo fixo para o navegador; ou podem ser gerados dinamicamente, ou seja, o servidor Web permite que programas sejam executados através dele e gerem páginas com dados variáveis, por exemplo, dados de funcionários, tabelas, folha de pagamento etc.

Para que um programa sendo executado num servidor Web possa se comunicar de maneira eficiente com o navegador que está no micro de quem acessa a Internet, foi criado um padrão de comunicação, para evitar que cada programa se comunicasse de um jeito com o servidor e com o navegador. Esse padrão foi denominado de CGI (Common Gateway Interface).

Portanto, o objetivo da Datamace é criar programas em Cobol seguindo o padrão CGI, que acessem bases de dados Cobol e transmitam informações para o navegador do internauta, devidamente autorizado para acessar determinadas funções.

Requisitos

Para que seja possível executar um programa CGI, é necessário que haja um servidor Web na Intranet ou na Internet.

Os servidores Web dependem do sistema operacional que está instalado no computador que é o servidor. Os principais sistemas operacionais utilizados nos computdores que são os servidores Web são Windows/NT, Windows2000 e Linux.

Para a plataforma Windows, o principal programa de servidor Web é o IIS (Internet Information Server) da própria Microsoft, que acompanha as versões completas do Windows NT e do Windows 2000. É o principal servidor Web do mercado.

Outra alternativa importante é o sistema operacional Linux, que também detém razoável parcela do mercado e utiliza o servidor Web Apache.

O NetExpress é capaz de compilar programas CGI/Cobol para a plataforma Windows. Contudo, para a plataforma Linux é preciso adquirir um outro produto (Server Objetcts) a fim de conseguir gerar o código executável que rode em Linux, mas utilizando o mesmo código fonte (segundo informações obtidas junto à DTS). Dessa forma, a princípio, o objetivo da Datamace é trabalhar com a plataforma Windows, e, posteriormente, verificar a viabilidade da aquisição do Server Objects para podermos trabalhar também com Linux.

Obs: Usando o Server Objects, o NetExpress gera o programa para rodar em Linux, mas provavelmente, os dados necessitarão estar em um servidor Windows ou Novell, pertencente à mesma rede interna do servidor Linux.

Possibilidades

Existem diferentes maneiras de montar uma arquitetura para o acesso via Web dos Sistemas Datamace

  • Na Intranet do Cliente : O cliente deve ter um servidor Web (plataforma Windows) e com isso, a infraestrutura de uma Intranet preparada (instalado o protocolo TCP/IP). As novas aplicações para Web darão acesso aos dados dos Sistemas Datamace através da Intranet, ou seja, somente quem estiver conectado à rede local terá acesso.
  • Na Internet em servidor Web do Cliente : Os dados ficam disponíveis para a Internet, através de login/senha e conexão segura. Mas o servidor onde estiverem os dados deverão pertencer à mesma rede do servidor Web.
  • Na Internet em servidor Web na Datamace: Os dados ficam na Datamace, disponíveis para a Internet, através de login/senha e conexão segura. Essa arquitetura (a Datamace ficando com os arquivos do cliente em servidores da Datamace) é denominada ASP (Application Service Provider). Tal configuração deverá ser implementada em prazo mais longo, pois além de prever possíveis problemas de segurança, a Datamace precisa possuir maior infraestrutura de link, servidores etc.
