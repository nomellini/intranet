Projetos : Segurança na Web

1) Sistemas em DOS rodando em Rede Local


Os sistemas Datamace já possuem hoje o conceito de login. No ambiente Web isso será mantido, fazendo a costumeira verificação da senha do usuário registrada no cadastro. A Datamace, ao gravar a senha do usuário não a grava de forma "aberta", ou seja, ela não fica gravada como ela é. Existem técnicas que permitem utilizar-se de cálculos matemáticos (algorítmos) onde os caracteres gravados ficam "inintelegíveis" para quem não conhece o cálculo que foi feito. De forma geral, essas técnicas são conhecidas como "criptografia". Além disso, os formatos de armazenamento dos campos numéricos dos arquivos Cobol exigem certo conhecimento técnico para serem entendidos, se acessados diretamente pelos arquivos.

Contudo, estando em rede local, a princípio, não é necessária uma grande preocupação com uma possível interceptação do conteúdo enquanto o mesmo trafega pela rede. Isso porquê, em tese, numa rede local estão apenas usuários locais. Dessa forma, o atual modelo de segurança dos sistemas Datamace em DOS atende as necessidades das empresas de forma geral.

2) Sistemas rodadando em ambiente Web

Tratando-se de ambiente Web existem outras preocupações. Além de o servidor Web precisar ser protegido por programas específicos (conhecidos como Fire-Walls), para proteger da invasão de hackers, ainda é necessário se preocupar com a interceptação de conteúdo. Em outras palavras, não basta criptografar a senha na hora de gravá-la em arquivo, é preciso criptografar todos os dados antes de enviá-los, tanto no caminho servidor/navegador como no caminho contrário. Para atender esse tipo de necessidade, existem as chaves de criptografia. Tais chaves tornam mais ou menos complexas a tarefa de "quebrar" o algorítmo e conseguir decifrar o conteúdo dos dados criptografados. Uma criptografia de 40 bits (o tamanho da chave é medido pela quantidade de bits que a compõe) é considerada "fraca", enquanto uma de 128 bits é tida como "forte". Chaves maiores do que essas (40 bits ou 128 bits) os EUA não permitem que sejam exportadas, ou seja, somente eles detém os maiores segredos de segurança do mundo.

Para utilizar-se dessa segurança, um servidor Web precisa ser "certificado" por um Orgão de Certificação ("Certification Authority" - CA), a qual, mediante o pagamento de uma anuidade, concede ao servidor a possibilidade de efetuar uma "conexão segura" (você já observou ao fazer uma transação pela Internet quando aquele pequeno cadeado no rodapé do navegador fica fechado ? ). Estabelecendo uma conexão segura, o servidor negocia com o navegador de quem acessa o site uma criptografia para ambos "conversarem". A partir desse momento, só os dois computadores poderão "entender" os dados que trafegam entre eles.
Essa é a tecnologia utilizada, por exemplo, pelos bancos que possuem Internet Banking.

3) As necessidades

Portanto, para efetuar uma conexão segura, um servidor precisa ser certificado por uma CA. Como a própria Datamace deseja ter a possibilidade de hospedar um site o qual permita conexão segura, nosso servidor será certificado (isso está sendo pesquisado e providenciado).
No caso de nossos sistemas ambiente Web serem hospedados em servidor do próprio cliente (o que deve acontecer na maioria dos casos), o cliente torna-se resposável pela certificação de seu servidor.

Independentemente da conexão, a vamos manter a nossa senha criptografada no arquivo Cobol, criando assim mais um nível de sergurança.

Acrescentada a tudo isso, existe ainda a segurança do próprio Web server, o qual pode exigir, conforme a configuração, que haja um login em sua rede antes de permitir o acesso à determinada área do site.
